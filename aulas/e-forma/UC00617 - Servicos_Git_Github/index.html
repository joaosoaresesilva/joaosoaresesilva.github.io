Plano de Sessão: UC00617 - Utilizar Serviços Git e GitHub

Duração Total: 25 Horas (1500 minutos)

Público-alvo: Desenvolvedores, estudantes e qualquer profissional interessado em versionamento de código e colaboração.

Modalidade: E-learning (Aulas assíncronas com vídeos, leituras, exercícios práticos e síncronas para dúvidas e workshops).

Objetivos da Formação:

  • Compreender os conceitos fundamentais de controle de versão e do Git.
  • Dominar os comandos Git essenciais para o dia a dia de desenvolvimento.
  • Utilizar o GitHub para hospedagem de repositórios, colaboração e gestão de projetos.
  • Aplicar boas práticas de versionamento de código.
  • Resolver conflitos de merge de forma eficiente.
  • Configurar e gerenciar chaves SSH para autenticação segura.

Módulo 1: Introdução ao Controle de Versão e Git (3 Horas - 180 minutos)

1.1. O que é Controle de Versão? (30 minutos)

  • História e importância do controle de versão.
  • Tipos de sistemas de controle de versão (VCS): Local, Centralizado (CVCS) e Distribuído (DVCS).
  • Vantagens de usar um sistema de controle de versão.

1.2. Introdução ao Git (45 minutos)

  • O que é Git e por que usá-lo.
  • Filosofia e arquitetura do Git.
  • Instalação e configuração inicial do Git (usuário e e-mail).

1.3. O Ciclo de Vida do Arquivo no Git (45 minutos)

  • Área de Trabalho (Working Directory), Área de Staging (Staging Area/Index) e Repositório Local (Local Repository).
  • Estados dos arquivos: Untracked, Unmodified, Modified, Staged.

1.4. Comandos Básicos do Git - Parte 1 (60 minutos)

  • git init: Inicializando um novo repositório.
  • git add: Adicionando arquivos à área de staging.
  • git commit: Registrando mudanças no histórico.
  • git status: Verificando o estado do repositório.
  • git log: Visualizando o histórico de commits.

Módulo 2: Trabalhando com o Git Localmente (6 Horas - 360 minutos)

2.1. Desfazendo Mudanças no Git (90 minutos)

  • git restore: Descartando mudanças no working directory.
  • git reset: Desfazendo commits e removendo arquivos da área de staging.
  • git rm: Removendo arquivos do controle de versão.
  • git mv: Movendo/renomeando arquivos.

2.2. Entendendo e Gerenciando Branches (120 minutos)

  • O que são branches e por que usá-las.
  • git branch: Listar, criar e deletar branches.
  • git checkout: Navegar entre branches.
  • Boas práticas de branching (e.g., branch principal, feature branches).

2.3. Merge e Rebase (90 minutos)

  • git merge: Unindo histórico de branches.
  • Tipos de merge (Fast-forward e Non-fast-forward).
  • Conflitos de merge: identificação e resolução manual.
  • git rebase: Reorganizando o histórico de commits.
  • Diferenças e quando usar merge vs. rebase.

2.4. Tags e Stash (60 minutos)

  • git tag: Marcando versões específicas do projeto.
  • git stash: Armazenando temporariamente mudanças não comitadas.

Módulo 3: Introdução ao GitHub e Colaboração (8 Horas - 480 minutos)

3.1. Conceitos Fundamentais do GitHub (60 minutos)

  • O que é GitHub e sua importância na comunidade de desenvolvimento.
  • Estrutura de um repositório no GitHub.
  • Perfil de usuário, organizações e repositórios.
  • Escolhendo a visibilidade do repositório (público vs. privado).

3.2. Conectando o Git ao GitHub (90 minutos)

  • Criação de um repositório no GitHub.
  • git remote add: Adicionando um repositório remoto.
  • git push: Enviando commits para o repositório remoto.
  • git pull: Atualizando o repositório local com mudanças do remoto.
  • git clone: Clonando um repositório existente.
  • Configuração de chaves SSH para autenticação (geração e adição ao GitHub).

3.3. Fluxo de Trabalho Colaborativo com Pull Requests (120 minutos)

  • Forking de repositórios.
  • Criação e gerenciamento de Pull Requests (PRs).
  • Revisão de código e comentários em PRs.
  • Merge de Pull Requests.
  • Resolvendo conflitos em Pull Requests.

3.4. Gerenciamento de Issues e Projetos (90 minutos)

  • Criação e rastreamento de Issues (problemas, bugs, melhorias).
  • Atribuição de Issues, rótulos e marcos (milestones).
  • Quadros de projetos (Projects) no GitHub para gestão de tarefas.
  • GitHub Actions (breve introdução ao CI/CD).

3.5. Contribuindo para Projetos Open Source (120 minutos)

  • Ética e diretrizes de contribuição.
  • Encontrando projetos para contribuir.
  • Fluxo de trabalho de contribuição (fork, clone, feature branch, commit, push, pull request).

Módulo 4: Tópicos Avançados e Boas Práticas (5 Horas - 300 minutos)

4.1. Refatorando o Histórico de Commits (90 minutos)

  • git amend: Editando o último commit.
  • git rebase -i: Rebase interativo para editar múltiplos commits (squash, reword, drop).
  • Perigos do rebase em histórico público.

4.2. Hooks do Git (60 minutos)

  • O que são Git Hooks.
  • Exemplos de hooks comuns (pre-commit, post-commit).
  • Automação de tarefas com hooks.

4.3. Ignorando Arquivos com .gitignore (30 minutos)

  • Criando e configurando o arquivo .gitignore.
  • Melhores práticas para ignorar arquivos.

4.4. Boas Práticas e Fluxos de Trabalho (90 minutos)

  • Convenções de mensagens de commit (Conventional Commits).
  • Fluxos de trabalho populares (Git Flow, GitHub Flow, GitLab Flow).
  • Gerenciamento de conflitos avançado.
  • Uso de GUIs para Git (opcional).

4.5. Considerações Finais e Próximos Passos (30 minutos)

  • Recursos adicionais para aprendizado contínuo.
  • Comunidades e fóruns de suporte.
  • Apresentação de projetos finais (se aplicável ao formato do curso).

Módulo 5: Revisão e Projeto Prático Final (3 Horas - 180 minutos)

5.1. Sessão de Perguntas e Respostas e Revisão (60 minutos)

  • Revisão dos tópicos mais complexos.
  • Esclarecimento de dúvidas gerais.

5.2. Projeto Prático Final (120 minutos)

  • Criação de um repositório do zero, aplicando todos os conceitos aprendidos.
  • Simulação de um fluxo de trabalho colaborativo com branches, merges e pull requests.
  • Resolução de conflitos em um cenário prático.
  • (Opcional) Implementação de um GitHub Action simples.

Duração Total da Formação: 1500 minutos (25 Horas)
